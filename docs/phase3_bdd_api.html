1. Schéma de Base de Données — Vue d'Ensemble

Le schéma repose sur PostgreSQL 16 avec l'extension TimescaleDB. Toutes les tables de données portent un tenant_id pour l'isolation multi-tenant. Le champ updated_at est mis à jour automatiquement via un trigger.

2. TimescaleDB — Hypertable & Continuous Aggregates

Table / VueRésolutionRétentionUsage frontend
metrics (hypertable)~60s (brut)30 joursGraphiques temps réel ≤ 6h
metrics_1h (continuous agg)1 heure365 joursGraphiques historiques 6h–7j
metrics_1d (continuous agg)1 jour730 jours (2 ans)Graphiques long terme > 7 jours
Règle de résolution automatique (côté backend Go) :
Plage ≤ 6h → metrics (brut) · 6h–7j → metrics_1h · > 7j → metrics_1d
Le handler metric lit la plage from/to et route la requête vers la bonne table.

4. Spécification des Routes API REST

Conventions :

  • Préfixe : /api/v1
  • Auth : JWT Bearer RS256 (sauf routes publiques)
  • tenant_id extrait du JWT — jamais dans l'URL
  • Succès : {"data": ..., "meta": {...}}
  • Erreur : {"error": {"code": "...", "message": "..."}}
  • Pagination : cursor-based (?cursor=&limit=, défaut 50, max 200)
