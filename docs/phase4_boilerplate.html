4. Mesure CPU Linux — Méthode Différentielle

La mesure CPU via /proc/stat nécessite deux lectures séparées dans le temps. L'agent stocke le snapshot précédent dans un état statique et calcule la différence à chaque collecte :

usage (%) = (total_delta - idle_delta) / total_delta × 100

où :
  total_delta = Σ(user+nice+system+idle+iowait+irq+softirq+steal) entre t-1 et t
  idle_delta  = (idle + iowait) entre t-1 et t
Premier appel : comme il n'y a pas de snapshot précédent au démarrage, leo_metrics_init() effectue une première lecture de /proc/stat pour initialiser l'état. La première collecte retournera des valeurs valides dès le premier appel à leo_metrics_collect().

5. Thread-Safety de la File d'Envoi WSS

Thread heartbeat / Thread métriques
        │
        ▼  leo_conn_send(data, len)
┌───────────────────────────────────────────┐
│  pthread_mutex_lock(&queue_mutex)         │
│  send_queue[queue_tail] = {data, len}     │
│  queue_tail = (queue_tail+1) % DEPTH      │
│  queue_count++                            │
│  pthread_mutex_unlock(&queue_mutex)       │
│  lws_cancel_service(lws_ctx)  ← réveille │
└───────────────────────────────────────────┘
        │
        ▼  Thread LWS (event loop)
┌───────────────────────────────────────────────────┐
│  LWS_CALLBACK_CLIENT_WRITEABLE                    │
│    pthread_mutex_lock(&queue_mutex)               │
│    frame[LWS_PRE..] = send_queue[queue_head].data │
│    lws_write(wsi, frame+LWS_PRE, len, TEXT)       │
│    queue_head++; queue_count--                    │
│    if queue_count > 0: lws_callback_on_writable() │
│    pthread_mutex_unlock(&queue_mutex)             │
└───────────────────────────────────────────────────┘
